SIRIUS ConfigDB Service

 

It is service for saving ans restoring SIRIUS control system configurations based on Mongo backend database and a Flask/nginx-uwsgi frontend.

The source code is located at https://github.com/lnls-sirius/docker-config-db

There is a python package implemented to interact with this servive: https://github.com/lnls-sirius/dev-packages/tree/master/siriuspy/siriuspy/clientconfigdb

Currently an instance of the service runs at http://10.0.38.42/config-db

Examples of implemented frontend routes:

  1. Current services stats: http://10.0.38.42/config-db/stats
  2. List of currently defined  configuration types (default route): http://10.0.38.42/config-db/configs
  3. List of currently saved configurations of a certain type ("global_config", in this example): http://10.0.38.42/config-db/configs/global_config
  4. List of currently discarded configurations of a certain type ("global_config", in this example): http://10.0.38.42/config-db/configs/discarded/global_config
  5. Get configuration of a certain type ("global_config" here) by its name ("Temp2", in this example): http://10.0.38.42/config-db/configs/global_config/Temp2

If the configuration type or the configuration name has a special character ('(', ')', ' ', and so on...) the URL need to be encoded. The following online service can be used:

https://onlinehtmleditor.dev/
